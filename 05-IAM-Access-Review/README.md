# Day 5 — IAM Access Review

## Purpose

This folder documents the IAM and access review work completed for the Nexus Cloud Operations Lab.

The goal was to practise reviewing access assignments and applying least privilege thinking before moving into Azure Storage RBAC and access testing.

## Scenario

Nexus Retail Group needed a basic access review to confirm that access was being managed intentionally and that users or groups were not being given unnecessary permissions.

## Work Completed

- Reviewed the purpose of Azure IAM access control
- Reviewed the difference between user-based and group-based access
- Reviewed least privilege access principles
- Planned access validation for Finance and Sales users
- Prepared the access control structure used in the later Storage RBAC testing

## Access Review Focus Areas

- Who should have access
- What role should be assigned
- Whether access should be assigned to users or groups
- Whether Finance access should be restricted to Finance users
- Whether Sales users should be excluded from Finance resources
- Whether disabled users should remain blocked from active access

## Skills Demonstrated

- IAM access review thinking
- Least privilege access planning
- Group-based access control planning
- Role assignment awareness
- Access validation preparation
- Security documentation
