# Day 15 — Jira SLA and MSP Support Structure

## Purpose

This folder documents the Jira SLA and MSP support structure for the Nexus Cloud Operations Lab.

The goal was to make the service desk more realistic by adding priority thinking, ticket status review, escalation logic, and SLA-style support documentation.

## Scenario

Nexus Retail Group needed a more structured support process after Jira Service Management was selected and set up.

The support desk needed clear expectations for how tickets should be prioritised, tracked, escalated, and resolved.

## Work Completed

- Reviewed ticket priority structure
- Defined SLA-style thinking for support requests
- Planned escalation workflow
- Planned MSP-style ticket handling
- Documented how support tickets should move through the service desk
- Prepared the support structure for later Active Directory, backup, monitoring, and endpoint management tickets

## SLA and Priority Focus

The support structure considered:

- High priority incidents
- Medium priority service requests
- Low priority documentation tasks
- Access-related tickets
- Offboarding-related tickets
- Backup and monitoring tickets
- Escalation path for unresolved issues

## Skills Demonstrated

- SLA awareness
- Ticket priority planning
- Escalation planning
- MSP support process documentation
- Service desk operations planning
- IT support workflow design
