# Day 11 — Log Analytics and KQL

## Purpose

This folder documents the Log Analytics and KQL monitoring work completed in the Nexus Cloud Operations Lab.

The goal was to practise reviewing Azure activity and monitoring data using Log Analytics and basic KQL queries.

## Scenario

Nexus Retail Group needed visibility into Azure activity so that support and operations work could be reviewed, queried, and documented.

## Work Completed

- Reviewed Log Analytics workspace
- Practised basic KQL queries
- Queried Azure activity data
- Reviewed resource activity and status information
- Captured monitoring evidence
- Documented the monitoring review process

## KQL Focus Areas

The KQL work focused on basic operational visibility, including:

- Azure activity review
- Resource activity checks
- Status review
- Monitoring evidence collection

## Skills Demonstrated

- Log Analytics workspace navigation
- Basic KQL usage
- Azure activity review
- Monitoring investigation
- Operational evidence capture
- Cloud support documentation
