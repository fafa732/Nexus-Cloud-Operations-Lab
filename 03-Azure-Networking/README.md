# Day 3 — Azure Networking

## Purpose

This folder documents the Azure networking setup for the Nexus Cloud Operations Lab.

The goal was to create a basic network structure that could support identity, server, storage, monitoring, and support workloads in Azure.

## Scenario

Nexus Retail Group needed a structured Azure network before deploying and supporting cloud resources.

This networking foundation helped separate resources logically and prepared the environment for later server and Active Directory work.

## Work Completed

- Created the core Azure virtual network
- Created subnets for different workload areas
- Created network security group structure
- Reviewed subnet association
- Captured networking evidence for documentation

## Network Areas

The lab network included subnet planning for areas such as:

- Corporate resources
- Server workloads
- Security or DMZ-style separation
- Active Directory/server expansion

## Skills Demonstrated

- Azure virtual network creation
- Subnet planning
- Network security group review
- Basic cloud network segmentation
- Azure portal navigation
- Infrastructure documentation
