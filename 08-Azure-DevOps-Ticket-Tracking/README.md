# Day 8 — Azure DevOps Ticket Tracking

## Purpose

This folder documents the early ticket-tracking phase of the Nexus Cloud Operations Lab.

The goal was to practise documenting support-style work using Azure DevOps Boards before Jira Service Management became the official service desk platform later in the project.

## Scenario

Nexus Retail Group needed a way to track support tasks, access requests, and security validation work.

Azure DevOps Boards was used during the early project phase to organise tickets and document progress.

## Work Completed

- Created support-style work items
- Organised tickets by status
- Documented access-related tasks
- Tracked Finance storage access work
- Tracked Sales access validation work
- Tracked disabled user/offboarding review
- Captured early service desk evidence

## Ticket Areas

The early tickets covered:

- Finance storage access request
- Ex-employee account review
- Sales access validation
- Finance RBAC verification
- RBAC testing evidence documentation

## Skills Demonstrated

- Ticket tracking
- Support task documentation
- Work item management
- Access request documentation
- Ticket status tracking
- Service desk workflow awareness
