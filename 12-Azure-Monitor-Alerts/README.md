# Day 12 — Azure Monitor Alerts

## Purpose

This folder documents the Azure Monitor alerting work completed in the Nexus Cloud Operations Lab.

The goal was to practise creating alerting evidence so that resource activity and potential issues could be monitored instead of only reviewed manually.

## Scenario

Nexus Retail Group needed basic monitoring and alerting so that support staff could identify issues and respond through a documented process.

## Work Completed

- Reviewed Azure Monitor
- Created or reviewed an action group
- Created an alert rule
- Reviewed alert status
- Captured alerting evidence
- Documented monitoring and alerting workflow

## Monitoring Focus Areas

The alerting work focused on:

- Resource monitoring
- Alert rule setup
- Action group awareness
- Alert status review
- Operational response documentation

## Skills Demonstrated

- Azure Monitor navigation
- Alert rule creation
- Action group configuration
- Monitoring evidence capture
- Cloud operations awareness
- Support documentation
